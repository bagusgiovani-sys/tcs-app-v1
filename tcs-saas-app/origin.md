## CLAUDE.md Violations
- `components/admin/ProductForm.tsx` exists — CLAUDE.md says "confirm before adding anything not in spec." This component was never wired up.
- Dark mode accent token (`#CE9760`) diverges from light mode accent (`#F4B942`). CoffeeRadarChart hardcodes `#F4B942` — will look wrong in dark mode.
- `<a href="/loyalty">` in ProfileClient violates App Router convention (should be `<Link>`).

## Critical Findings (must-fix)

1. **Cart badge permanently hidden** — `BottomNav.tsx:152` animates `scale: 0` on both `initial` and `animate`. Badge is always invisible even when cart has items.
2. **Missing `users` table DB trigger** — `auth.ts` register creates auth user but never inserts into `users` table. All `supabase.from('users').select('name')...` queries return null. Patched in some places with `user_metadata.name` fallback but breaks loyalty page customer name display, profile page, and admin order queue customer names.
3. **POS `handleConfirm` missing shop_id filter** — `POSClient.tsx:86` updates order without `.eq('shop_id', SHOP_ID)`. Identical bug to the 3 we already fixed in AdminOrderQueue.

---

## Major Findings (should-fix)

4. **`components/admin/ProductForm.tsx` is dead code** — Callback-based API, not connected to any page. `AdminProductForm.tsx` is the real one. This will confuse any future contributor.
5. **`nextjs-splash/` directory in repo root** — Template copy of splash components. Not imported anywhere. Pollutes repo, should be deleted.
6. **Sidebar links `/admin/vouchers` and `/admin/loyalty` return 404** — Pages don't exist. Admin clicking these gets a broken page.
7. **Profile "Riwayat Pesanan" is a dead `<div>`** — `ProfileClient.tsx:100–112` renders a row that looks tappable but has no `href` or `onClick`. Does nothing.
8. **Admin dashboard revenue query fetches all rows** — `admin/page.tsx:13–15` does `.select('total')` then JS `.reduce()`. Should use Supabase aggregate `.select('total.sum()')`.
9. **`<a>` instead of `<Link>` in ProfileClient** — `ProfileClient.tsx:115`. Full page reload on tap.

---

## Minor Findings (nice-to-fix)

10. **CoffeeRadarChart hardcodes hex colors** — `#F4B942` and `rgba(244,185,66,0.22)` inline. Won't adapt to dark/vibe mode. Should use `var(--color-brand-accent)`.
11. **FOUC script unguarded** — `layout.tsx:48` accesses `localStorage` without try/catch. Private browsing / strict browsers will throw, breaking page render.
12. **POS order_items insert no error handling** — `POSClient.tsx:67` — same orphan-order bug fixed in checkout. Missing rollback.
